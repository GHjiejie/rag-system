# 02｜Apple Silicon 与配置选择

本篇用于回答“芯片代际怎么选”“内存和存储该配多少”等问题。

## Apple Silicon 的核心特性

- CPU、GPU、神经网络引擎一体化设计。
- 统一内存架构：各计算单元共享内存池。
- 在能效比上通常表现优秀，日常体验流畅。

## 配置优先级建议

在预算有限时，建议优先级如下：

1. 内存（统一内存）
2. 芯片档位
3. 存储容量

> 原因：内存不足对多任务和长期使用体验影响最直接。

## 内存怎么选

| 使用强度 | 建议内存 |
|---|---|
| 轻办公/学习 | 16GB |
| 开发（多 IDE + 浏览器 + 容器） | 16GB~24GB |
| 视频剪辑/设计/多任务重载 | 24GB 起 |

## 存储怎么选

| 使用习惯 | 建议存储 |
|---|---|
| 主要云端文档、轻量本地文件 | 512GB |
| 经常存素材、虚拟机镜像、工程缓存 | 1TB 起 |

## 开发者专项建议

- 使用 Docker、数据库、多个仓库并行开发：优先加内存。
- 如果项目依赖本地大模型或大体积数据：内存与存储都要提高档位。
- 使用外置高速硬盘可缓解存储压力，但不能替代内存。

## 芯片代际选择思路

- 不必盲目追新代；先看你的真实负载。
- 同代高内存配置，往往比“新一代低内存”更实用。
- 若需要更强持续性能和接口能力，可考虑 Pro 系列机型。

## FAQ

### Q1：8GB 还能买吗？
仅适合非常轻量用途。若追求 3~5 年稳定体验，建议至少 16GB。

### Q2：统一内存和传统内存有什么体感区别？
多任务场景下更高效，但内存不足时同样会出现卡顿。

### Q3：存储不够后期能否补？
机身内部通常不可自行升级，建议购买时考虑未来 2~3 年需求。
